# localmind walkthrough

What you'd see when installing and running localmind on a fresh Linux
laptop with 16 GB of RAM and no discrete GPU. This is the textual twin of
`scripts/demo.gif`; it exists for slow connections, terminal browsers,
and people who'd rather read than watch a recording.

## 1. Install (5 seconds)

```
$ curl -fsSL https://raw.githubusercontent.com/bsaisuryacharan/localmind/main/install.sh | sh
==> installing localmind 0.0.6 (linux/amd64) to /home/you/.localmind
==> downloading https://github.com/bsaisuryacharan/localmind/releases/download/v0.0.6/localmind-linux-amd64.tar.gz
==> verifying checksum: ok
==> installed: /home/you/.local/bin/localmind
==> next: run `localmind init` to configure, then `localmind up`
```

`localmind` is a single static Go binary (the wizard). Nothing else got
installed at the system level.

## 2. Configure (2 seconds)

`localmind init` profiles the host and writes a `.env` next to the compose
file. On a 16 GB CPU-only box it picks the `cpu_low` profile, which maps
to `qwen2.5:3b` for chat and `nomic-embed-text` for embeddings (see
`models.yml`).

```
$ localmind init
detected: os=linux arch=amd64 ram=16GB gpu="" vram=0GB
picked profile: cpu_low
wrote /home/you/localmind/.env
next: `localmind up`
```

The generated `.env`:

```
# generated by `localmind init`
LOCALMIND_PROFILE=cpu_low
WEBUI_PORT=3000
OLLAMA_PORT=11434
WHISPER_PORT=9000
PIPER_PORT=10200
MCP_PORT=7800
WEBUI_AUTH=true
```

## 3. Bring the stack up (warm: ~20s, cold: 3-10 min)

```
$ localmind up
[+] Running 6/6
 ✔ Network localmind_default        Created
 ✔ Container localmind-ollama-1     Started
 ✔ Container localmind-open-webui-1 Started
 ✔ Container localmind-whisper-1    Started
 ✔ Container localmind-piper-1      Started
 ✔ Container localmind-mcp-1        Started
==> profiler: running quick ollama benchmark (skip with --no-profile)
==> profiler: qwen2.5:3b first-token 480ms, sustained 22 tok/s
==> profiler: result written to .localmind/profile.json
```

On cold boot the compose layer also pulls the chat + embedding models the
first time `ollama` starts; the profiler waits for them. Subsequent
`localmind up` runs reuse the cached weights and the profiler short-circuits
(it sees an existing `.localmind/profile.json`).

## 4. Open the WebUI

The WebUI is at `http://localhost:3000`. First load shows the
Open WebUI sign-up screen (the first registered user becomes the admin,
because the wizard set `WEBUI_AUTH=true`). After signing in, you pick a
model from the dropdown — `qwen2.5:3b` is already pulled and visible.

A first chat turn:

```
You:    explain MCP in one sentence.
qwen:   Model Context Protocol (MCP) is an open protocol that lets AI
        assistants discover and call external tools, data sources, and
        services through a single standardized interface.
```

Latency on a CPU-only box is roughly 500 ms to first token, then
~20 tok/s sustained for a 3B-parameter model.

## 5. Talk to the responder (mobile path, optional)

The responder is the host-side service that answers HTTP even when the
docker stack is asleep. After `localmind responder install`:

```
$ curl -s http://localhost:7900/status
{"stack_running":true,"webui_url":"http://localhost:3000","responder":"ok"}
```

Hitting `/wake` when the stack is already up returns immediately:

```
$ curl -s -X POST http://localhost:7900/wake
{"already_running":true,"webui_url":"http://localhost:3000"}
```

When the stack is down, `/wake` runs `docker compose up -d` and waits for
the WebUI to answer before responding:

```
{"woke":true,"webui_url":"http://localhost:3000"}
```

If wake-up exceeds the timeout you'll see:

```
{"woke":false,"error":"timed out waiting for stack","webui_url":"http://localhost:3000"}
```

This is the indirection that makes the Tailscale Funnel URL stable: your
phone always reaches the responder, and the responder transparently boots
the heavy bits on demand.

## 6. Stop the stack

```
$ localmind down
[+] Running 6/6
 ✔ Container localmind-mcp-1        Removed
 ✔ Container localmind-piper-1      Removed
 ✔ Container localmind-whisper-1    Removed
 ✔ Container localmind-open-webui-1 Removed
 ✔ Container localmind-ollama-1     Removed
 ✔ Network localmind_default        Removed
```

Volumes are preserved — your chats, RAG index, and pulled models survive.

That's the whole happy path: one install line, one configure step, one
up, one chat. Everything else (`backup`, `restore`, `responder`,
`tunnel`, `profile`) layers onto this baseline without changing it.
